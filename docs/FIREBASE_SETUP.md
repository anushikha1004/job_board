# Firebase Setup Guide

Complete guide to integrate Firebase Firestore with your IT Jobs Board application.

## Quick Setup (5 minutes)

### Step 1: Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Click **"Add project"**
3. Enter project name: `it-jobs-board`
4. Accept defaults and click **Create project**
5. Wait for project creation to complete

### Step 2: Create Web App

1. In Firebase Console, click the web icon `</>`
2. Register app with name: `it-jobs-board-web`
3. Copy the Firebase config (you'll need this)
4. Click **Continue to console**

### Step 3: Set Environment Variables

1. In your project, create or edit `.env.local` file:

```bash
cp .env.local.example .env.local
```

2. Open `.env.local` and fill in your Firebase config:

```env
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key_here
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id
```

### Step 4: Create Firestore Database

1. In Firebase Console, go to **Build** → **Firestore Database**
2. Click **Create database**
3. Choose location: **(us-central1)** recommended
4. Start in **Test mode** (for development)
5. Click **Create**

### Step 5: Create Jobs Collection

1. In Firestore, click **+ Start collection**
2. Collection ID: `jobs`
3. Click **Next**
4. For first document, enter:
   - Document ID: `job1`
   - Add fields as shown below

### Sample Job Document Structure

Add this sample job to test:

```json
{
  "title": "Senior DevOps Engineer",
  "company_name": "Acme Tech",
  "description": "Lead our infrastructure team and design scalable cloud architectures",
  "tags": ["Python", "Docker", "Kubernetes", "AWS"],
  "location": "Remote",
  "type": "Full-time",
  "apply_url": "https://example.com/apply",
  "salary_range": {
    "min": 120000,
    "max": 180000,
    "currency": "USD"
  },
  "created_at": (current server timestamp),
  "updated_at": (current server timestamp)
}
```

**Important:** For the timestamp fields:
- Click on the value field
- Select "Server timestamp" from the dropdown
- Firebase will auto-fill the current time

### Step 6: Test the Connection

1. Restart your dev server:
```bash
npm run dev
```

2. Open http://localhost:3000
3. You should see your job from Firebase!

If still showing demo data:
- Check browser console (F12) for errors
- Verify all env variables are correct
- Check Firestore security rules (see Security Rules section below)

## Firestore Security Rules

This project now includes production-oriented Firestore rules in:

- `firestore.rules`
- `firebase.json` (points Firestore to that rules file)

The rules enforce:

- `user_profiles` can only be read/written by the profile owner
- `jobs` are public-read, but create/update/delete only by authenticated users with role `company`
- job mutations are restricted to the original poster (`postedBy == request.auth.uid`)
- `company_profiles` are public-read, but only company owners can create/update their own profile
- `favorites` can only be read/written by the authenticated owner

To apply security rules:

1. Install Firebase CLI if needed: `npm i -g firebase-tools`
2. Login: `firebase login`
3. Select your project: `firebase use <project-id>`
4. Deploy rules: `firebase deploy --only firestore:rules`

If you prefer manual setup, copy the contents of `firestore.rules` into **Firestore Database → Rules** and publish.

## Data Schema

### Jobs Collection

```typescript
interface Job {
  id: string;                    // Auto-generated by Firestore
  title: string;                 // Job title (e.g., "Senior DevOps Engineer")
  company_name: string;          // Company name
  description?: string;          // Job description
  tags: string[];                // Technologies (e.g., ["Python", "Docker"])
  location?: string;             // Location (e.g., "Remote", "On-site", "Hybrid")
  type?: string;                 // Job type (e.g., "Full-time", "Contract")
  apply_url: string;             // Application URL
  salary_range: {
    min: number;                 // Min salary in cents (e.g., 120000)
    max: number;                 // Max salary in cents (e.g., 180000)
    currency?: string;           // Currency code (e.g., "USD")
  };
  created_at: Timestamp;         // Auto-generated on creation
  updated_at: Timestamp;         // Auto-updated on changes
}
```

## Using Firebase in Code

### Fetch All Jobs

```typescript
import { getAllJobs } from '@/lib/firestore';

const jobs = await getAllJobs();
```

### Fetch Jobs by Tag

```typescript
import { getJobsByTag } from '@/lib/firestore';

const pythonJobs = await getJobsByTag('Python');
```

### Fetch Featured Jobs

```typescript
import { getFeaturedJobs } from '@/lib/firestore';

const featured = await getFeaturedJobs(3);
```

### Create New Job

```typescript
import { createJob } from '@/lib/firestore';

const newJob = await createJob({
  title: 'Backend Engineer',
  company_name: 'TechCorp',
  tags: ['Node.js', 'PostgreSQL'],
  salary_range: { min: 100000, max: 150000, currency: 'USD' },
  apply_url: 'https://example.com/apply',
  location: 'Remote',
  type: 'Full-time',
  description: 'Build robust APIs',
  created_at: new Date().toISOString(),
  updated_at: new Date().toISOString(),
});
```

### Update Job

```typescript
import { updateJob } from '@/lib/firestore';

await updateJob('job1', {
  title: 'Senior Backend Engineer',
  // ... other fields to update
});
```

### Delete Job

```typescript
import { deleteJobById } from '@/lib/firestore';

await deleteJobById('job1');
```

## Using Firebase Hooks

### useFirestoreJobs Hook

```typescript
import { useFirestoreJobs } from '@/lib/hooks';

function MyComponent() {
  // Fetch jobs on mount
  const { jobs, isLoading, error, refetch } = useFirestoreJobs(true);

  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;

  return (
    <div>
      {jobs.map(job => (
        <div key={job.id}>{job.title}</div>
      ))}
      <button onClick={refetch}>Refresh</button>
    </div>
  );
}
```

## Common Issues

### ❌ "Missing Firebase configuration"

**Problem:** Environment variables not set or incorrect

**Solution:**
1. Check `.env.local` file exists
2. Copy-paste values from Firebase Console exactly (no spaces)
3. Restart dev server after changing env vars

### ❌ "PERMISSION_DENIED" errors

**Problem:** Firestore security rules are too restrictive

**Solution:**
1. Go to Firestore → Rules
2. Make sure rules allow read access:
```
allow read: if true;
```
3. Click Publish
4. Restart your app

### ❌ Jobs showing as demo data

**Problem:** Firebase is not connected or returning empty results

**Solution:**
1. Add sample jobs to Firestore manually
2. Check browser console (F12) for errors
3. Verify Firestore has a `jobs` collection with documents
4. Check network tab to see if requests are being made

### ❌ Timestamps appearing as incorrect dates

**Problem:** Firestore Timestamp not converting properly

**Solution:** Already handled in `lib/firestore.ts`:
```typescript
created_at: doc.data().created_at?.toDate?.().toISOString()
```

## Next Steps

### 1. Add Authentication
Use Firebase Authentication to allow only admins to post jobs:
```typescript
import { getAuth, signInWithEmailAndPassword } from 'firebase/auth';
```

### 2. Enable Real-time Updates
Replace `getAllJobs()` with real-time listeners:
```typescript
import { onSnapshot } from 'firebase/firestore';

onSnapshot(collection(db, 'jobs'), (snapshot) => {
  const jobs = snapshot.docs.map(doc => doc.data());
  setJobs(jobs);
});
```

### 3. Deploy to Production
1. Update security rules for production
2. Set up environment variables on Vercel
3. Deploy with `npm run build && npm start`
